Retrieval Mode
BM25 saturates repeated terms and normalises for chunk length. Default in Elasticsearch and Lucene.
4-step RAG pipeline

1. Ingest: text split into 500-word chunks with 80-word overlap
2. Index: TF-IDF and BM25 matrices built in memory
3. Retrieve: top-4 chunks scored and returned
4. Generate: passages sent to gemini-2.0-flash

TF-IDF vs BM25
TF-IDF uses cosine similarity on term-frequency vectors. BM25 adds saturation and length normalisation, the default in production search engines.
