4-step RAG pipeline:

1. Ingest — text split into 500-word chunks with 80-word overlap
2. Index — TF-IDF matrix built in memory; no vector DB required
3. Retrieve — cosine similarity scores your question against all chunks; top-4 returned
4. Generate — retrieved passages sent to claude-haiku-4-5 as context; grounded answer returned

Why TF-IDF?
Zero infrastructure, fully explainable, sub-millisecond retrieval. Works well for financial text where terminology is consistent (revenue, margin, EPS appear verbatim). Dense embeddings would add semantic recall at the cost of a vector DB dependency.
